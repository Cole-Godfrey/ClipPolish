# Versioning Policy

ClipPolish uses Semantic Versioning (`MAJOR.MINOR.PATCH`).

## Rules

- `MAJOR`: incompatible API or behavior changes.
- `MINOR`: backward-compatible feature additions.
- `PATCH`: backward-compatible fixes.

## Pre-1.0 Behavior

While major version is `0`, minor versions may include breaking changes.
Breaking changes must still be clearly documented in `CHANGELOG.md`.

## Tagging

- Release tags use `vMAJOR.MINOR.PATCH` (example: `v1.0.0`).
- Every release tag must have a matching changelog entry.
